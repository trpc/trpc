tRPC is not a server itself, but rather a library that needs to be integrated with a host system. **Adapters** act as the bridge between tRPC and your chosen hosting platform, allowing you to deploy your API anywhere.

<Callout title="Choose Your Platform">
  tRPC adapters follow common conventions but are tailored for specific hosting
  environments. Pick the adapter that matches your deployment target.
</Callout>

## Common Patterns

All tRPC adapters share common configuration patterns:

### Basic Setup

```typescript
import { createAdapter } from '@trpc/server/adapters/[adapter-name]';
import { appRouter } from './router';
import { createContext } from './context';

const handler = createAdapter({
  router: appRouter,
  createContext,
  onError: ({ error, type, path, input, ctx, req }) => {
    console.error('tRPC Error:', error);
    // Send to error reporting service
  },
});
```

### Context Creation

Each adapter provides request/response objects in context creation:

```typescript
export function createContext({ req, res }) {
  return {
    req,
    res,
    user: getUserFromRequest(req),
    db: prisma,
  };
}
```

### Error Handling

Handle errors globally across your API:

```typescript
const handler = createAdapter({
  router: appRouter,
  createContext,
  onError: ({ error, type, path, input, ctx, req }) => {
    if (error.code === 'INTERNAL_SERVER_ERROR') {
      // Send to monitoring service
      console.error('ðŸ’¥ Internal server error:', error);
    }
  },
});
```

## Choosing an Adapter

<Accordions>
  <Accordion title="For Development">
    **Standalone Server** is perfect for getting started quickly. It requires no
    additional setup and provides a simple HTTP server.
    
    ```bash
    npm create trpc-app@latest my-app --template standalone
    ```
  </Accordion>

<Accordion title="For Full-Stack Apps">
  **Next.js** provides the best full-stack experience with built-in API routes,
  SSR support, and seamless client integration. Features: - App Router and Pages
  Router support - Built-in API routes - Server-side rendering - Static site
  generation
</Accordion>

<Accordion title="For Serverless">
  **AWS Lambda** or **Cloudflare Workers** for edge computing and serverless
  deployments. Benefits: - Automatic scaling - Pay-per-request pricing - Global
  edge locations - No server management
</Accordion>

  <Accordion title="For Existing APIs">
    **Express** or **Fastify** if you have an existing Node.js API and want to add tRPC endpoints.
    
    Integration:
    - Add tRPC to existing routes
    - Share middleware and authentication
    - Gradual migration path
  </Accordion>
</Accordions>

## Platform Compatibility

| Platform           | Adapter                  | Runtime        | Notes                         |
| ------------------ | ------------------------ | -------------- | ----------------------------- |
| Next.js            | `nextjs`                 | Node.js / Edge | App Router & Pages Router     |
| Vercel             | `nextjs` / `fetch`       | Node.js / Edge | Recommended: Next.js adapter  |
| Netlify            | `express` / `fetch`      | Node.js        | Use Netlify Functions         |
| Railway            | `express` / `standalone` | Node.js        | Full Node.js support          |
| Render             | `express` / `standalone` | Node.js        | Full Node.js support          |
| AWS Lambda         | `aws-lambda`             | Node.js        | Serverless functions          |
| Cloudflare Workers | `cloudflare-workers`     | V8             | Edge runtime                  |
| Deno Deploy        | `fetch`                  | Deno           | Use fetch adapter             |
| Bun                | Any                      | Bun            | Compatible with most adapters |

## Migration Guide

### From Express REST API

```typescript
// Before: Express route
app.get('/api/users/:id', async (req, res) => {
  const user = await getUserById(req.params.id);
  res.json(user);
});

// After: tRPC procedure
export const appRouter = router({
  getUserById: publicProcedure
    .input(z.object({ id: z.string() }))
    .query(async ({ input }) => {
      return await getUserById(input.id);
    }),
});
```

### From GraphQL

```typescript
// Before: GraphQL resolver
const resolvers = {
  Query: {
    user: async (parent, { id }, context) => {
      return await context.db.user.findUnique({ where: { id } });
    },
  },
};

// After: tRPC procedure
export const appRouter = router({
  getUser: publicProcedure
    .input(z.object({ id: z.string() }))
    .query(async ({ input, ctx }) => {
      return await ctx.db.user.findUnique({ where: { id: input.id } });
    }),
});
```
